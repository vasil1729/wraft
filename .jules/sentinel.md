## 2024-05-18 - [Fix timing attack in admin login]
**Vulnerability:** The admin login endpoint (`SessionController.create/2`) evaluated the `is_deactivated` flag of a user before performing password verification. This early exit introduced a timing attack vector, allowing an attacker to enumerate the deactivated status of administrative accounts. Furthermore, if a user lookup failed, the application did not call `Bcrypt.no_user_verify()`, enabling basic user enumeration through timing discrepancies.
**Learning:** In authentication flows using Elixir's `with` statement, placing business logic checks (like account status flags) before the computationally expensive password verification creates timing leaks. Short-circuiting execution reveals whether those earlier conditions were met based on the request's response time.
**Prevention:** Always verify passwords *before* checking other business logic rules (such as `is_deactivated`) within authentication procedures. Ensure `Bcrypt.no_user_verify()` is explicitly called on paths where a user is not found (e.g., catching `nil` in the `else` block) to simulate hashing delays uniformly.
