## 2023-10-27 - [CRITICAL] Fixed Gnuplot Command Injection and Path Traversal in Gantt Chart Generator
**Vulnerability:** Gnuplot CLI arguments (like `title`) and `.csv` input paths were directly interpolated into a `.plt` script string. Since Gnuplot's execution evaluates its scripts, injecting double quotes, backslashes, or backticks can allow OS command execution or break syntax. Furthermore, user uploads were written directly to `temp/` using their unsanitized `filename`, leading to path traversal, and hardcoded `temp/` paths caused race conditions for concurrent executions. The `System.cmd` execution also bypassed secure standard libraries in favor of shelling out to copy files.
**Learning:** External tool integrations like Gnuplot are often overlooked as injection vectors. Gnuplot is notoriously capable of shell execution (`system()` or backtick execution), making input escaping inside `.plt` strings extremely critical. Furthermore, hardcoded temp directories shared across processes in a Phoenix app lead to unsafe multi-user access.
**Prevention:** Always escape backslashes, double quotes, and backticks when embedding strings in Gnuplot scripts. Use `Path.basename/1` combined with a regex allowlist (`~r/[^a-zA-Z0-9_\-\.]/`) to prevent path traversal with user-provided filenames. Always isolate temporary generation artifacts into unique directories (e.g., using `Ecto.UUID.generate()`) within `System.tmp_dir!()`, and ensure cleanup happens in a `try...after` block. Avoid `System.cmd("cp", ...)` in favor of `File.cp!`.
