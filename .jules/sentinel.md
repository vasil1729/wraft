## 2024-05-24 - [Timing Attacks and User Enumeration in Elixir Auth]
**Vulnerability:** Timing attacks and user enumeration possible in `signin` and `session` controllers due to early `with` statement short-circuiting on user lookups, skipping password verification. Password reset flows also risked enumeration.
**Learning:** `with` macro short-circuiting can leak information about user existence via timing differences (bypassing slow `Bcrypt.verify_pass/2`). Using `Bcrypt.no_user_verify/0` on invalid lookups simulates hashing delay. However, putting `Bcrypt.no_user_verify/0` in unhappy paths of operations that normally DO NOT verify passwords (like password reset generation) introduces a CPU-exhaustion DoS vulnerability.
**Prevention:** 1. Refactor auth `with` statements into nested `if`/`case` to guarantee `Bcrypt.no_user_verify/0` is called on invalid user lookups, and ensure `verify_pass` is checked BEFORE business logic flags (like `is_deactivated`). 2. Never use `Bcrypt.no_user_verify/0` in unhappy paths of flows that do not normally hash passwords (e.g. password resets); just return generic success.
