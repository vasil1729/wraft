## 2024-06-13 - Fix User Enumeration in Password Reset
**Vulnerability:** User Enumeration via Password Reset Endpoint. The `generate_token` endpoint previously returned a 404 error ("No user with this email.!") when an invalid email was submitted, but a 200 OK ("Success") for valid emails. This allowed attackers to enumerate registered users on the platform.
**Learning:** We must *not* use `Bcrypt.no_user_verify()` in password reset flows (where no password checking is normally performed on the happy path) to simulate delays. Because the happy path only generates a token and sends an email without verifying a password, forcing a bcrypt hash on the unhappy path introduces a severe CPU-exhaustion Denial of Service (DoS) vulnerability. Both paths should be fast and identical.
**Prevention:** Always ensure that authentication endpoints return standardized, generic error messages or success messages regardless of the input's validity, and avoid introducing unnecessary expensive operations on failure paths unless they are mirroring an expensive operation on the success path.
