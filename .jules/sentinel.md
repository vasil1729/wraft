## 2024-05-24 - [Fix user enumeration timing attack in admin session controller]
**Vulnerability:** A timing attack in the admin session login (`SessionController.create/2`) allowed attackers to enumerate valid internal user email addresses, as well as their deactivation status. Using `with` combined with `is_deactivated: false` matching skipped `Bcrypt.verify_pass/2` entirely for invalid emails or deactivated users, resulting in noticeably different response times.
**Learning:** In Elixir authentication flows, checking business logic flags (like `is_deactivated`) before verifying passwords, or short-circuiting password verification via `with` statements when user lookup fails, creates a timing side-channel.
**Prevention:** Always use nested `if` statements for authentication flows to guarantee that `Bcrypt.verify_pass/2` is evaluated exactly once for valid users, and explicitly call `Bcrypt.no_user_verify/0` when a user lookup fails to simulate password hashing delay.
