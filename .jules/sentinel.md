## 2024-05-24 - Fix User Enumeration via Timing Attacks in Auth Controllers
**Vulnerability:** User enumeration via timing attacks during sign in flows due to early returns on non-existent users without simulating password hash delays. Additionally, in `SessionController`, `is_deactivated` status was checked before password validation, leaking account status to unauthorized users. `UserController` also leaked email existence by allowing `invalid_email` error to propagate to the fallback controller.
**Learning:** In Elixir `with` statements used for authentication, failing early on user lookup or checking business flags prior to password validation opens up timing side-channels and enumeration vectors. `Bcrypt.no_user_verify()` must be invoked at the controller level on `nil` or `invalid_email` responses to neutralize timing leaks.
**Prevention:** Always verify passwords before evaluating business logic flags (like active/deactivated status), and explicitly simulate password hashing time with `Bcrypt.no_user_verify()` when a user lookup fails. Return generic error messages for both non-existent users and invalid passwords.
