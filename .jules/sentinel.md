## 2024-05-24 - [Timing Attack and User Enumeration in Session Controller]
**Vulnerability:** The admin login flow in `SessionController.create` checked if an account was deactivated *before* verifying the user's password, and failed to call `Bcrypt.no_user_verify()` when an email didn't exist. This allowed attackers to enumerate valid email addresses based on response time differences (timing attack) and discover if valid accounts were deactivated without needing a password.
**Learning:** In Elixir authentication flows, short-circuiting `with` statements upon failed user lookups or checking status flags before verifying passwords can cause severe information leaks.
**Prevention:** Always verify the user's password (`Bcrypt.verify_pass`) *before* checking business logic status flags like `is_deactivated`. Ensure a catch-all clause for failed lookups explicitly calls `Bcrypt.no_user_verify()` to simulate password hashing delays. Avoid double-hashing by only calling it when the user lookup explicitly fails.
