## 2024-05-24 - [Fix command injection and path traversal in Gnuplot SVG generator]
**Vulnerability:** Command injection and Path Traversal were possible because `filename` and `title` variables derived from a user-uploaded file were not sanitized before being copied via `System.cmd("cp", ...)` and injected into a generic temporary location, and the `title` was passed unescaped to a `.plt` file executed by `gnuplot`.
**Learning:** Elixir `System.cmd` is susceptible to OS command injection. Gnuplot script injection acts like command injection if bad strings are not escaped. Unique directories (with UUIDs) should be used instead of shared, static `temp` locations to prevent data races and info leaks.
**Prevention:** Always use `File.cp!` instead of `System.cmd("cp", ...)` for file operations. Use `System.tmp_dir!()` and `Ecto.UUID.generate()` to isolate temporary environments. Escape variables correctly for `gnuplot` strings (escaped `\`, `"`, and `` ` ``). Sanitize dynamically constructed filenames using `Path.basename()` and regex filters `~r/[^a-zA-Z0-9_\-]/`. Clean up via `try/after`.
