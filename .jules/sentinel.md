## 2025-01-29 - [Fix Timing Attack in Admin Authentication]
**Vulnerability:** The Elixir `with` statement in `SessionController.create` short-circuited on invalid user lookup or deactivated status, bypassing the expensive `Bcrypt.verify_pass/2` call. This caused responses for non-existent users or deactivated users to be significantly faster, enabling user enumeration via timing attacks.
**Learning:** Using `with` for authentication flows in Elixir can easily introduce timing attacks if failure paths (like missing users) skip computationally heavy operations.
**Prevention:** Avoid `with` statements for password verification logic. Prefer nested `if` or `case` statements to guarantee that computationally expensive operations like `Bcrypt.verify_pass/2` are evaluated exactly once on the happy path, and `Bcrypt.no_user_verify/0` is explicitly invoked on the unhappy path (e.g., when a user is not found). Also, ensure password validation occurs *before* evaluating internal status flags like `is_deactivated` to prevent enumeration of account states.
