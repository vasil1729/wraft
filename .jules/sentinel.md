## 2025-03-09 - [Command Injection via OS Command Execution in Gnuplot Gantt Chart]
**Vulnerability:** The `generate_gnu_gantt_chart/2` function executed user-supplied string data directly via the OS command line without escaping or sanitization, resulting in a severe OS Command Injection vulnerability. Furthermore, a predictable directory structure was used which leads to a path traversal vulnerability and race conditions on concurrent executions.
**Learning:** `System.cmd/3` only escapes spaces but does not prevent logic within string replacement that is used via interpreted scripts such as `.plt` scripts. If `.plt` scripts interpret commands or variables incorrectly or via command substitution, it can trigger injected code execution.
**Prevention:** Use strictly managed outputs and temporary unique paths via `Ecto.UUID.generate()`, clean up paths using `try/after` block, sanitize file inputs using `Path.basename/1`, and strictly escape string payloads (like titles) manually for tools like Gnuplot using `String.replace` specifically matching Gnuplot injection chars.
