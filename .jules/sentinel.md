## 2024-04-14 - Command Injection via Gnuplot Script Generation
**Vulnerability:** Command injection was possible because unsanitized strings (like file paths and chart titles based on user input) were inserted directly into a Gnuplot script (`.plt`) via `String.replace`. Gnuplot allows executing OS commands if special characters (like backticks) are present in string literals. In addition, there was a risk of path traversal via the uploaded `filename`.
**Learning:** String interpolation or replacement into script files (like `.plt`, `.sh`) effectively acts as OS command injection if the strings are not properly escaped or sanitized. Furthermore, concurrent requests generating Gantt charts could cause race conditions since temporary files were written to a static directory.
**Prevention:** Always escape backslashes, quotes, backticks, and newlines when dynamically generating scripts. Use `Path.basename()` and restrict allowed characters to sanitize uploaded filenames and other user inputs used in file paths. Generate unique temporary directories (e.g., using `Ecto.UUID.generate()`) and use `try/after` blocks to ensure cleanup, preventing resource leaks and race conditions.
