## 2024-05-18 - [Preventing Timing Attacks in Elixir Authentication Flows]
**Vulnerability:** In `SessionController`, early exit patterns (`with` statements) checked an `is_deactivated` flag before performing password verification (`Bcrypt.verify_pass()`). Additionally, failed user lookups (`InternalUsers.get_by_email()`) did not execute a computationally equivalent password hash.
**Learning:** Early exits prior to computationally expensive operations (like Bcrypt) create timing side-channels that allow attackers to enumerate user email existence and specific account statuses. Password verification MUST ALWAYS happen before evaluating any business logic flags. Failed lookups MUST simulate the same CPU load using `Bcrypt.no_user_verify()`.
**Prevention:** Avoid `with` statements for authentication login paths as they naturally encourage short-circuiting on failure, leading to timing attacks if not handled correctly in the `else` block. When refactoring, ensure only one slow execution path exists (either verify actual hash or simulate hash) and evaluate account flags strictly after a successful password check.
