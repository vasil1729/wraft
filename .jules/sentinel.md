## 2024-05-18 - [CRITICAL] Command Injection, Race Conditions, and Path Traversal in Template Scripts
**Vulnerability:** The `generate_gnu_gantt_chart/2` method had multiple critical vulnerabilities. It concatenated user input (`filename`, `title`) unsafely into file paths and Gnuplot scripts without sanitization. It wrote to a hardcoded predictable `temp/` directory leading to race conditions. It used `System.cmd("cp", ...)` to move files leading to shell injection via `filename` and command injection inside Gnuplot itself.
**Learning:** External templates processed by CLI utilities (like Gnuplot) are susceptible to command injection if variables aren't properly escaped (like escaping `` ` ``, `\`, `"`). Hardcoded shared `temp/` directories are extremely unsafe in concurrent web applications. File copying must use standard language libraries (`File.cp!`) instead of relying on OS shelling.
**Prevention:** Always use `Path.basename` to prevent directory traversal in file uploads. Isolate temporary files in UUID-based directories in `System.tmp_dir!()`. Escape strings passed to templates targeting CLI processes. Use built-in Elixir `File` API instead of `System.cmd` for basic file operations.
