## 2025-02-14 - Fix Path Traversal and Command Injection in Gantt Chart Generation
**Vulnerability:** The `generate_gnu_gantt_chart` function suffered from path traversal via unsanitized uploaded filenames, race conditions/collisions from using hardcoded temporary directory paths, and Gnuplot command injection via unsanitized strings injected into templates. It also leaked temporary files.
**Learning:** Even internal tool invocations like Gnuplot require proper escaping. Unsanitized file paths are common sources of path traversal vulnerabilities. Using `Ecto.UUID.generate()` with `System.tmp_dir!()` is essential to prevent race conditions in temporary file operations, but these directories must be cleaned up to prevent denial of service (disk space exhaustion).
**Prevention:** Always use `Path.basename()` and restrict character sets for user-supplied filenames. Apply strict escaping (e.g., escaping backslashes, double quotes, and backticks) before injecting strings into script templates. Generate temporary files in isolated UUID-based directories and ensure they are cleaned up after use.
