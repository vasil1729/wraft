## 2024-03-13 - [HIGH] Fix user enumeration via timing attacks
**Vulnerability:** User enumeration via timing attack in `wraft_doc_web/controllers/session_controller.ex` and `wraft_doc_web/controllers/user_controller.ex`.
**Learning:** Elixir's `with` statement can short-circuit authentication flow when a user is not found, returning immediately without executing password hashing operations. Since password hashing with Bcrypt is intentionally slow, an attacker can determine if an email exists by measuring the time it takes for the server to respond. If the response is fast, the user does not exist. If the response is slow, the user exists but the password was incorrect.
**Prevention:** To prevent this, when an authentication flow short-circuits due to a missing user, always explicitly call `Bcrypt.no_user_verify()` in an `else` block to simulate the delay of password hashing. Furthermore, ensure that the error message returned to the user is a generic message mapping, such as `{:error, :invalid}`, so that both valid and invalid email lookups respond in similar time and with similar responses.
