## 2024-06-03 - [Fix Timing Attack & Enum Vulnerability in SessionController]
**Vulnerability:** User enumeration via logic flaws and timing attacks in authentication `with` blocks. Specifically, `SessionController.create` would instantly exit on a failed user lookup (revealing email non-existence via timing) or instantly exit on a deactivated user check (revealing account status without a password check).
**Learning:** `with` clauses that exit early upon failed condition checks can introduce timing leaks or logic bypasses in authentication flows. Elixir developers often place status checks (like `is_deactivated`) *before* password checks inside `with` statements, unintentionally allowing status enumeration.
**Prevention:** Avoid `with` blocks for critical authentication paths. Prefer nested `if` statements that guarantee the computationally expensive operation (`Bcrypt.verify_pass/2`) happens *before* checking business status logic (e.g. `is_deactivated`). Use `Bcrypt.no_user_verify/0` on failed user lookups to simulate hashing delay.
