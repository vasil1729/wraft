## 2024-03-15 - [User Enumeration Timing Leak]
**Vulnerability:** User Enumeration via timing attacks during authentication.
**Learning:** In Elixir, short-circuiting `with` statements upon failed user lookups causes a timing leak because password verification hashing (using Bcrypt) takes significantly longer. If an email address is not found, the response returns quickly; if the email is found but the password is wrong, the response is delayed. An attacker can use this timing discrepancy to enumerate valid emails on the system.
**Prevention:** To prevent User Enumeration, use `Bcrypt.no_user_verify()` when a user lookup fails. It's crucial to apply this simulation delay at the controller level during the auth flow, rather than inside generic context lookup functions like `Account.find/1`, to avoid potential Denial of Service (DoS) vulnerabilities if lookups are heavily used elsewhere in the application. Standardize error responses (like returning `{:error, :invalid}`) to prevent enumeration through different error messages.
