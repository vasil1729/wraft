## 2024-05-10 - Command Injection & Path Traversal in Gnuplot Script Generation
**Vulnerability:** The `generate_gnu_gantt_chart/2` function was vulnerable to OS command injection and path traversal because user-controlled `filename` and `title` variables were directly interpolated into a `.plt` script string which was then executed using `gnuplot`. Additionally, the lack of UUID-based temporary file isolation and cleanup led to race conditions and storage leaks.
**Learning:** Shell or command injection can occur not just through direct arguments to `System.cmd/2` (which are list-based and safe in Elixir), but indirectly when user input is interpolated without escaping into a script file that is subsequently executed by a system interpreter like `gnuplot`.
**Prevention:** Always strictly escape variables interpolated into `.plt` files (or similar script files). Use a restricted regex to sanitize filenames (e.g., `~r/[^a-zA-Z0-9_\-\.\ ]/`). For file generation, ensure temporary directories are unique (using `Ecto.UUID.generate()`) and strictly cleaned up using `try/after` and `File.rm_rf!`.
